# Portfolio Web – Diseño y Desarrollo de Interfaces (DOR)

## Descripción del proyecto

Este proyecto consiste en el desarrollo de un **portfolio web personal**, cuyo objetivo es presentar de forma clara y profesional la información académica, experiencia laboral, habilidades técnicas y proyectos realizados.

El enfoque principal del trabajo está en el **diseño de la interfaz**, la **experiencia de usuario (UX)** y la **maquetación mediante Tailwind CSS**.

---

## Objetivos del proyecto

- Diseñar una **interfaz web clara, usable y coherente**
- Aplicar un **proceso de diseño de interfaces web**
- Implementar una estructura visual con **jerarquía clara**
- Garantizar un diseño **responsive**
- Aplicar criterios básicos de **accesibilidad**
- Utilizar **Tailwind CSS** como herramienta principal de diseño
- Integrar la interfaz en una aplicación web funcional

---

## Tecnologías utilizadas

### Tailwind CSS (Elemento central del proyecto)

Tailwind CSS se ha utilizado como **herramienta principal de diseño y maquetación**, permitiendo:

- Aplicar estilos mediante **clases utilitarias**
- Mantener una **paleta de colores consistente**
- Crear componentes reutilizables (cards, botones, layouts)
- Implementar diseño **responsive** mediante breakpoints
- Mejorar la mantenibilidad y legibilidad del código
- Separar claramente estructura, estilo y contenido

El uso de Tailwind permite centrarse en el **diseño de la interfaz** sin necesidad de hojas CSS tradicionales, lo que encaja con los objetivos del módulo DOR.

### Vue.js

Vue.js se ha utilizado para:

- Estructurar la aplicación en **componentes**
- Gestionar la navegación entre vistas
- Facilitar la reutilización de elementos de interfaz

El uso de Vue no es el foco principal del proyecto, ya que su aprendizaje pertenece a otras asignaturas del ciclo, pero sirve como base para integrar la interfaz diseñada.

## Diseño visual y elección de la paleta de colores

### Paleta utilizada

- **Fondo principal:** `slate-900`
- **Fondos secundarios (cards, secciones):** `slate-800`
- **Texto principal:** `slate-100`
- **Texto secundario:** `slate-400 / slate-300`
- **Color de acento:** `sky-400 / sky-500`

### Justificación de la elección

Se ha elegido una paleta basada en tonos **azul grisáceo (slate)** combinados con un **color acento azul (sky)** por tratarse de colores comúnmente asociados al ámbito tecnológico y profesional.

Esta combinación:

- Transmite **sobriedad, profesionalidad y modernidad**
- Evita colores excesivamente saturados que puedan distraer
- Permite establecer una **jerarquía visual clara**
- Mejora la **legibilidad** gracias al contraste entre fondo oscuro y texto claro
- Reduce la fatiga visual en sesiones de lectura prolongadas

El color de acento se utiliza de forma controlada para destacar:
- Títulos principales
- Enlaces
- Elementos interactivos
- Acciones importantes (botones)

---

## Estructura y jerarquía visual

El contenido del portfolio se organiza en distintas vistas:

- **Sobre mí**
- **Proyectos**
- **Habilidades**
- **Experiencia**
- **Contacto**

Cada sección se presenta mediante **tarjetas (cards)** con fondo secundario, bordes suaves y espaciado adecuado, lo que permite:

- Separar visualmente la información
- Facilitar la lectura
- Guiar al usuario por el contenido
- Mantener coherencia en toda la aplicación

Los títulos, subtítulos y textos secundarios utilizan tamaños y colores diferenciados para reforzar la jerarquía visual.

---

## Diseño responsive

El diseño se ha planteado siguiendo un enfoque **mobile-first**, utilizando las utilidades responsive de Tailwind (`sm`, `md`, `lg`).

- En dispositivos móviles, el contenido se adapta a una disposición vertical
- En pantallas grandes, se aprovecha el espacio mediante grids y layouts horizontales
- El objetivo es mantener la **usabilidad y legibilidad** en cualquier dispositivo

---

## Usabilidad y accesibilidad

Durante el diseño de la interfaz se han tenido en cuenta criterios básicos de accesibilidad y usabilidad:

- Uso de **HTML semántico**
- Contraste adecuado entre texto y fondo
- Tamaños de fuente legibles
- Espaciado suficiente entre elementos
- Navegación clara mediante una barra superior fija
- Indicadores visuales en elementos interactivos (hover, enlaces activos)

---

## Proceso de desarrollo aplicado (DOR)

El desarrollo del portfolio ha seguido las fases habituales del diseño de interfaces web:

1. **Análisis**
   - Identificación del usuario objetivo (reclutadores, profesorado, profesionales del sector)
   - Definición del contenido necesario

2. **Diseño**
   - Elección de paleta de colores
   - Definición de jerarquía visual
   - Diseño de componentes reutilizables

3. **Implementación**
   - Maquetación con Tailwind CSS
   - Integración en componentes Vue
   - Aplicación de diseño responsive

4. **Revisión**
   - Mejora de legibilidad
   - Ajustes de contraste y espaciado
   - Optimización de la experiencia de usuario
